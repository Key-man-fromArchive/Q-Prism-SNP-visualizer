## 1. 배경 (Background)

Q-Prism SNP Visualizer는 ASG-PCR SNP 판별을 위한 웹 분석 도구로, 기능적으로는 성숙해 있다(10개 최상위 탭, 다중 마커, 폴리플로이드, 다양한 벤더 포맷 임포트, 백엔드 테스트 통과). 그러나 UI/UX는 기능 확장 속도를 따라가지 못했고, 최근 화면 실측 결과 **"고장 난 것처럼 보이는"** 수준의 시각 결함이 확인되었다.

핵심 원인은 개별 화면의 실수가 아니라 **공통 토대의 부재**다. 재사용 가능한 *제네릭* UI 컴포넌트 계층(Button/Card/Modal/Field 등)이 없어 화면마다 스타일을 즉흥적으로 재작성했고(버튼 클래스 문자열 87곳, 인라인 `style` 98곳), 그 결과 일관성·다크모드·국제화·반응형이 화면별로 제각각 깨진다.

### 1.1 실측 근거 (현재 소스 빌드 기준, 2026-07-24)

현재 `main` 소스를 격리 컨테이너로 빌드해 12개 화면을 캡처·검증했다. 대표 결함:

- **한국어에서 헤더 툴바 붕괴**: export 버튼(실행취소·다시실행·인쇄·새파일·프로젝트)이 badge 필 폭 부족으로 **한 글자씩 세로 줄바꿈**되어 헤더 전체가 깨져 보인다.
- **핵심 화면이 백지**: Analysis의 대표 패널 "대립유전자 판별"(산점도)이 데이터가 있어도 로딩/빈/에러 상태 없이 **흰 여백**만 노출된다.
- **다크모드 툴바 붕괴**: export 필들이 다크 헤더 위에서 밝은 흰 블록으로 남는다(토큰 미적용).
- **아이콘 글자 깨짐(□)**: 이모지/HTML 엔티티/유니코드를 아이콘으로 사용해, 이모지 폰트가 없는 환경에서 업로드 히어로·다크모드 토글·경계선/분석 버튼 등이 두부(□)로 렌더된다.

### 1.3 아키텍처 사실관계 (요구사항이 반드시 준수해야 할 현행 동작)

- **Analysis 화면은 2개 렌더 경로다.** `AnalysisWorkspace`가 세션의 저장 마커 수로 분기한다: 마커 0개 → 단일마커 뷰(`AnalysisTab`의 2×2 그리드 + "마커로 분할" 배너), 마커 ≥1개 → `MultiMarkerAnalysisPanel`(마커별 판정). Analysis 관련 모든 요구사항·수용 기준은 **두 경로 모두**를 다룬다.
- **인증 2모드.** (a) `local`(자체 로그인), (b) `asg_launch`(ASG Designer 런치 토큰). `asg_launch`에서는 로컬 로그인 비활성, 실패/만료 시 전용 "ASG로 돌아가기" 화면(`App.tsx`), 헤더의 "ASG 저장" 버튼은 **런치 모드이면 노출**되고 `snp:save_result` 스코프가 없으면 **비활성(disabled)** 이며 saving/saved/error/dirty-reset 상태 머신을 가진다(`Header.tsx`). 개편은 이 렌더/비활성 규칙을 숨기거나·강제 활성화·상태를 리셋해서는 안 된다.
- **i18n 메커니즘.** 네임스페이스 프레임워크가 아니라 **타입드 flat 번역 객체 훅**이다: `hooks/use-i18n.ts` + `locales/en.ts`·`locales/ko.ts` + 영속화된 `stores/language-store.ts`. 문자열 추가는 두 로케일 파일에 대응 키를 넣고 언어 스토어 영속성을 보존해야 한다.
- **전역 상태는 다수의 Zustand 스토어로 분산.** `selection-store`(선택 웰·현재 사이클·재생), `session-store`(세션/업로드), `data-store`(클러스터·경계·오프셋), `settings-store`(영속 설정), `auth-store`, `language-store`. 컨트롤 재배치는 상태 소유권을 바꾸지 않아야 하며, 크로스-패널 동기화(플레이트↔산점도↔결과표)를 회귀시키면 안 된다(§7.1 상태 소유권 맵 참조).
- **테스트 현황.** 루트/`frontend/e2e`에 Playwright e2e가 있으나 **프런트 단위/컴포넌트 테스트 프레임워크는 없다**(`frontend/package.json`에 test 스크립트 없음). 백엔드는 pytest 보유.

## 2. 문제 정의 (Problem Statement)

> 기능은 충분하지만, **첫인상과 신뢰도**가 UI 결함으로 훼손된다. 실험 데이터를 다루는 과학 도구임에도 화면이 "미완성/고장" 인상을 주며, 특히 **한국어·다크모드**에서 두드러진다. 화면 간 불일치는 학습 비용과 오조작 위험을 높인다.

## 3. 목표 / 비목표 (Goals / Non-Goals)

### 3.1 목표

- G1. **"고장 나 보이는" 결함 제거**: 아이콘 □, 한국어 툴바 붕괴, 백지 패널, 다크모드 붕괴를 없앤다.
- G2. **일관성 확보**: 재사용 UI 프리미티브와 디자인 토큰으로 모든 화면이 하나의 시스템으로 읽히게 한다.

- G4. **접근성·국제화·반응형 기본선 충족**: 키보드/ARIA, 한/영 완전 대응, 주요 브레이크포인트 대응.

- N2. 신규 기능(신규 탭·신규 리포트) 추가. (기존 기능의 표현/배치만 개선)

## 4. 대상 사용자 & 사용 맥락 (Users & Context)

- 주 사용자: 분자진단/육종 연구자, 실험실 분석 담당자. qPCR·유전형 판별 도메인 지식 보유, UI 전문성은 낮음.
- 환경: 데스크톱 브라우저(주로 1440px 내외), 실험실 조명에서 라이트/다크 혼용. 한국어 우선, 영어 병행.
- 진입 경로: (a) `local` 자체 로그인, (b) `asg_launch` 런치. 두 경로 모두 동일 분석 UI를 사용하되 §1.3의 모드별 제약을 지킨다.
- 대표 과업: 파일 업로드 → 자동 분석/클러스터링 확인 → 웰 타입 보정 → 품질/통계 확인 → 결과 내보내기/(런치 모드)ASG 저장.

## 5. 디자인 원칙 (Design Principles)

- P1. **단일 시스템**: 색·간격·타이포·컴포넌트는 토큰과 프리미티브에서만 나온다. 화면에서 hex·임의 px 직접 지정 금지. **승인된 예외**: 데이터 구동 시각화 색(Plotly 시리즈, 웰 유전형/클러스터 색 매핑) — 단, 예외 목록에 명시된 것만.
- P2. **테마 안전**: 모든 UI는 라이트/다크에서 동일한 의미와 대비를 유지한다.
- P3. **로케일 안전**: 한국어에서 레이아웃이 깨지지 않는다(줄바꿈·폭 가정 금지, 전 문자열 i18n).
- P4. **상태 명시**: 데이터가 없거나 로딩·에러일 때 반드시 명시적 상태 UI를 보인다. 백지 금지.
- P5. **아이콘 일관**: 모든 *아이콘*은 lucide-react. 이모지·엔티티·유니코드 글자를 아이콘으로 쓰지 않는다. 단 **콘텐츠성 글자(단축키 글리프 등)는 아이콘이 아니므로 대상 아님**.
- P6. **점진·안전**: 기존 기능 회귀 없이 단계적으로 배포한다. 각 단계는 독립적으로 가치 있고 되돌릴 수 있다.
- P7. **기준점 상향**: 이미 잘 만들어진 화면(플레이트 설정)을 내부 품질 기준으로 삼아 나머지를 끌어올린다.
- P8. **셀렉터 안정성**: 기존 `id`/`data-testid`는 e2e 계약이다. 이관 시 보존하며, 불가피한 변경은 해당 테스트를 동시 수정한다.

## 6. 요구사항 (Requirements)

우선순위: **P0(필수·즉시)** > **P1(핵심)** > **P2(개선)**.

### 6.1 디자인 시스템 토대 (R1·R2·R3 / G2)

- FR-DS-1 (P1): `shared/ui/`에 제네릭 프리미티브 신설 — `Button`(variant: primary/secondary/ghost/danger, size: sm/md), `IconButton`, `Card`, `Field`+`Label`, `Modal`, `Badge`, `Toolbar`, `StatusState`(loading/error/empty), `Menu/Dropdown`. 기존 `shared/QcBadges`·`KeyboardHelpOverlay`는 삭제하지 않고 새 프리미티브 위에 재구성한다.
- FR-DS-1a (P0, 시드): Phase 0 의존 해소를 위해 **최소 프리미티브 시드**(`Button`, `IconButton`, `Menu/Dropdown`, `StatusState`)를 Phase 0에서 먼저 도입한다(§9 참조). 나머지 프리미티브는 Phase 1.
- FR-DS-2 (P1): 토큰 체계 정리. `index.css`의 `@theme` 토큰을 단일 소스로, 상태색(success/warning/danger/info)과 그 배경/보더 변형을 토큰으로 정의. 컴포넌트의 raw 팔레트(`text-green-600`, `bg-red-50` 등)·hex 직접 사용 제거(§5 P1 예외 제외).
- FR-DS-3 (P0): **미정의 CSS 변수 교정.** 대상: `var(--border)`·`var(--bg)`·`var(--primary)`·`var(--accent)`·`var(--text-muted)`(합계 ~15곳). **프로퍼티별 치환 매핑**을 명시한다 — 예: `borderColor: var(--border)` → `border-border` 클래스 또는 `var(--color-border)`; `background: var(--bg)` → `bg-bg`/`var(--color-bg)`; 색/텍스트도 동일 규칙. 구현계획에 전수 목록·치환표를 첨부한다.
- FR-DS-4 (P1): 타입 스케일 정리. `text-[8px]~[13px]` 임의 크기를 일관 스케일(xs/sm/base…)로 수렴. 밀집 그리드(플레이트 웰 라벨 포함) 최소 가독 크기 하한 규정.
- NFR-DS-5: 프리미티브는 접근성 기본을 내장한다. **`Modal` 완전 사양**: `role="dialog"` + `aria-modal="true"`, 접근 가능한 제목(`aria-labelledby`)·설명(`aria-describedby`), 오픈 시 초기 포커스(기본=1차 액션 또는 다이얼로그 컨테이너), 포커스 트랩, Esc 닫기, 명시적 취소(Cancel) 액션, 배경 `inert`/스크롤 락, **닫힐 때 호출 컨트롤로 포커스 복귀**. `Button`/`IconButton`은 포커스 링·`aria-label`(아이콘 전용 시)·disabled 시맨틱 내장.

### 6.2 헤더 & 툴바 (R1·R3·R4 / G1·G3)

- FR-HD-1 (P0): export 버튼(CSV/PNG/인쇄/PDF/XLSX)을 **"내보내기" `Menu`** 하나로 통합. 라벨은 아이콘+텍스트, 한국어에서 `whitespace-nowrap` + 충분한 폭 보장(세로 줄바꿈 금지).
- FR-HD-2 (P0): Undo/Redo는 `IconButton`(lucide `Undo2`/`Redo2`), disabled 상태 시각 명확화.
- FR-HD-3 (P1): 헤더를 영역으로 구획 — (좌) 브랜드+세션 컨텍스트(instrument/wells/cycles/QC 배지), (우) 액션·사용자·언어·테마.
- FR-HD-4 (P0): export/툴바 컨트롤을 토큰 기반으로 다크모드 정합화(밝은 블록 잔존 제거).
- FR-HD-5 (P1): 세션 컨텍스트 배지 과밀 시 우선순위 낮은 항목 요약/툴팁.
- FR-HD-6 (P0, 제약): ASG 저장 버튼은 **런치 모드이면 렌더**되고, `snp:save_result` 스코프가 없으면 **비활성(disabled)** 처리된다(현행 동작). 이 렌더/비활성 규칙과 saving/saved/error/dirty-reset 상태 머신, 링크드 컨텍스트 표시를 **그대로 보존**한다. 헤더 개편이 이를 변경하지 않음을 수용 기준에 포함.

### 6.3 내비게이션 & 레이아웃 재배치 (구조 / G3)

- FR-NAV-2 (P2): Analysis 내부 **세로 chrome 적층 축소** — 워크스페이스 서브탭 + 사이클 슬라이더 + Analyze/배수성/경계선 바를 **단일 스티키 분석 툴바**로 통합(3줄→1줄 지향). 단일마커/멀티마커 두 경로 모두 적용.
- FR-NAV-3 (P2): Analysis 2×2 그리드(산점도·플레이트·웰상세·결과표)를 반응형화(좁은 폭 1열 리플로우), 인라인 그리드 → 토큰/반응형 유틸.

### 6.6 아이콘·국제화·접근성·반응형 (횡단 / G1·G4)

- FR-X-3 (P1): 플레이트 그리드 접근성 통일 — `PlateView`도 실제 `button` 기반으로. **키보드 사양(확정)**: **로빙 tabindex 그리드** 채택(그리드 전체가 Tab 스톱 1개, 내부 이동은 화살표 ↑↓←→, Home/End=행 처음/끝). Enter/Space=선택 토글. **드래그 다중선택의 키보드 대체**: Shift+화살표로 선택 확장, 열/행 헤더 포커스 후 Enter=열/행 전체 토글, Esc=선택 해제. 포커스 가시성(포커스 링) 필수. 각 웰의 접근 가능한 이름/상태(예: "A1, 유전형 X, 선택됨"). `PlateSetupTab`도 동일 모델로 정렬.
- NFR-X-4 (P1): 텍스트/상태색 대비 WCAG AA 목표(라이트·다크 모두 검증). 자동 점검 도구(axe 등) 심각 위반 0.
- NFR-X-5: 반응형 하한(확정) — **1280px는 필수 지원(P1, 점진 이행)**: 화면별 이관 시점에 해당 화면의 1280px 본문 가로 오버플로 0을 게이트로 검증(Phase 1 이관 화면 → Phase 1 Exit, Analysis 2×2 그리드 → Phase 2 T2.3에서 마감). **태블릿 834px는 베스트-에포트(P2)**: 레이아웃이 깨지지 않을 것(치명적 겹침·잘림 없음)만 보장, 픽셀 완성도는 비목표. **예외 정책**: 384-well 플레이트·넓은 결과 테이블 등 본질적으로 넓은 콘텐츠는 자체 `overflow-x-auto` 컨테이너 안에서만 가로 스크롤 허용(페이지 본문은 스크롤 금지).

- M3 (G2): **일반 애플리케이션 액션 버튼**은 100% `Button`/`IconButton` 경유. **예외(시맨틱 컴포지트, 대상 아님)**: 탭 버튼, 플레이트 웰 셀, 메뉴 항목, Plotly modebar 등 서드파티/특수 인터랙션 컨트롤. 레이아웃/색 목적 인라인 `style={{`}`는 핵심 컴포넌트에서 제거(잔여는 데이터 구동 예외만).

- Q2. **산점도 백지 근인**(FR-ST-3): 구현계획에 **조사 태스크**로 포함. 데이터가 있는데 안 그려지는 버그로 판명되면 근인 수정, 아니면 상태 UI로 충분. 조사 결과 전에는 특정 버그 픽스를 확약하지 않는다.
